Part A.2: Recover Homographies

Because each pair is the result of a projective transform, image alignment can be performed by finding the transformation matrix (homography) between the pair of images:

 
    
    H = [[a, b, c],     // H = 3 x 3 homography matrix with 8 DOF
         [d, e, f],
         [g, h, 1]]
    
    p' = Hp             // projective transformation (p' = transformed image, p = original image)

For each image pair, the left image is the "original" image and the right image is the "transformed" image. Point correspondences are manually selected to help with alignment. These are stored in im1_pts and im2_pts, both of which have n points. For a single set of points, the following equation holds:

 
    
    a = im1_pts[i]                           // point in original image, (x, y)
    b = im2_pts[i]                           // point in transformed image, (x', y')
    
    a_arr = [    a[0],     a[1], 1].T
    b_arr = [w * b[0], w * b[1], w].T
    b_arr = H @ a_arr                        // this is the constraining equation

Expanding this equation (which, for now, is still for a single pair of points):

 
    
    w * b[0] = a * a[0] + b * a[1] + c
    w * b[1] = d * a[0] + e * a[1] + f
    w = g * a[0] + h * a[1] + 1

Substituting in w:

 
    
    w * b[0] = a * a[0] + b * a[1] + c
    (g * a[0] + h * a[1] + 1) * b[0] = a * a[0] + b * a[1] + c
    g * a[0] * b[0] + h * a[1] * b[0] + b[0] = a * a[0] + b * a[1] + c
    0 = a * a[0] + b * a[1] - g * a[0] * b[0] - h * a[1] * b[0] - b[0] + c

    w * b[1] = d * a[0] + e * a[1] + f
    (g * a[0] + h * a[1] + 1) * b[1] = d * a[0] + e * a[1] + f
    g * a[0] * b[1] + h * a[1] * b[1] + b[1] = d * a[0] + e * a[1] + f
    0 = d * a[0] + e * a[1] - g * a[0] * b[1] - h * a[1] * b[1] - b[1] + f

Each pair of point correspondences gives two equations:

 
    
    b[0] = a * a[0] + b * a[1] + c -                           g * a[0] * b[0] - h * a[1] * b[0]
    b[1] =                           d * a[0] + e * a[1] + f - g * a[0] * b[1] - h * a[1] * b[1]

Part A.3: Warp the Images

Now that each pair has a homography matrix, the images can be warped. This is done using inverse warping: for every pixel in the destination image (x', y'), first the corresponding source coordinates (x, y) are calculated, and then the source value f(x, y) is copied into g(x', y'). The case that (x, y) is non-integer, interpolation is used. The convention for interpolating pixel locations used here is setting (0, 0) to be the center of the top-left pixel. In order to ensure that the entire source image is warped, the bounding box of the transformed image is predicted ahead of time. For areas of the image that do not have valid values, the image remains black.

The first type of interpolation is nearest neighbor interpolation, where (x, y) is rounded to the nearest in-range pixel value. This produced the following result for the trees and building:

The second type of interpolation is bilinear interpolation, which takes the weighted average of the four surrounding pixel values based on the distance to each pixel. This produced the following images:

The tradeoff between the two interpolation methods is speed vs. quality. As can be seen above, the nearest neighbor interpolation results in a lower-quality output that suffers from aliasing, whereas the bilinear interpolation images are more smoothed and nice. A good example of this tradeoff is the roof of the building, which is grainy and aliased in the nearest neighbor image but smoothed in the bilinear image. However, the nearest neighbor implementation is almost 3x faster than the bilinear implementation, which take around ~2.5 sec and ~7 sec (respectively) to complete on average.
